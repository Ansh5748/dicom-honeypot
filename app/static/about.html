Project Overview

The DICOM Protocol Honeypot Analyzer is a specialized security tool designed to monitor, capture, and analyze unauthorized access attempts to medical imaging systems that use the DICOM (Digital Imaging and Communications in Medicine) protocol. By emulating a vulnerable PACS (Picture Archiving and Communication System) server, this honeypot attracts potential attackers and records their activities, providing valuable insights into attack patterns and techniques used against healthcare systems.

System Architecture

The system consists of several interconnected components:

  • DICOM Honeypot Server: Emulates a DICOM server with deliberate vulnerabilities to attract attackers
  • Elasticsearch: Stores all captured events and attack data for analysis
  • Kibana: Provides visualization and dashboards for monitoring attack patterns
  • API Server: Offers programmatic access to the collected data for integration with other security tools
  • Logstash: Processes and enriches log data before storing in Elasticsearch

Key Features

DICOM Protocol Emulation

Accurately emulates DICOM services including C-ECHO, C-FIND, C-MOVE, and C-STORE operations to appear as a legitimate PACS server. Supports standard DICOM transfer syntaxes and presentation contexts to ensure compatibility with various DICOM clients.

Honeytoken Generation

Creates synthetic patient records with embedded tracking identifiers (honeytokens) that serve as tripwires to detect data exfiltration attempts. Each honeytoken is uniquely identifiable, allowing the system to trace where and how the data was accessed.

Real-time Attack Monitoring

Provides real-time visibility into attack attempts through a comprehensive dashboard showing connection attempts, query patterns, and data access. Alerts on suspicious activities such as mass data retrieval, unusual query patterns, or access to honeytokens.

Comprehensive Logging

Records detailed information about each DICOM transaction including client IP, AE Title, command type, and full dataset contents. Maintains a complete audit trail of all system activities for forensic analysis and compliance reporting.

Implementation Details

The DICOM Honeypot is implemented using Python with the pynetdicom library for DICOM protocol support. The system is containerized using Docker and Docker Compose for easy deployment and scalability. The ELK stack (Elasticsearch, Logstash, Kibana) provides robust data storage, processing, and visualization capabilities.

Security Research Applications

This honeypot serves as a valuable tool for security researchers and healthcare organizations to:

  • Study attack patterns specific to medical imaging systems
  • Develop better protection mechanisms for patient data
  • Test DICOM security controls in a safe environment
  • Generate threat intelligence specific to healthcare systems
  • Understand compliance risks related to medical data access

Future Enhancements

Planned improvements to the system include:

  • Integration with threat intelligence platforms for IOC sharing
  • Support for additional DICOM service classes and transfer syntaxes
  • Enhanced visualization of attack patterns and data access attempts
  • Automated reporting for compliance with healthcare security frameworks
